APP 提交记录

今天提交APP, 时间 2017. 0706

在拥有证书的前提下, 按照adhoc的版本打包的样子, 在archive后, 选择第一个打包用于发布

这时, 需要本机有安装有这个找好下的发布证书. 打开XCode的 Account, 把主要帐号的profile都下下来, 就能顺利打包, 不然, 会印找不到证书而不能继续

完成后打开 application loader, 选择刚才打包的版本进行上传.

这时候, 可以看到在 “ 上传到 APPStore 审核 “这个步奏会很久, 据说是因为, 在上传时, 会在~目录下下载一个.itmstransporter文件夹, 大概有100m左右. 主要还是因为网络差, 有时候根本链接不上你也不知道. 用联通的网络会更顺利点. 用电信的则不行.

在 https://itunesconnect.apple.com/WebObjects/iTunesConnect.woa/ra/ng/app 新建一个App, 里面有对应的APP信息

可以查资料看需要填写什么信息. (注, 网上的教程可能会有过时, 还需要碰到不一样的地方注意细节)

(在开发中先创建好这个应用占坑. 这样可以获得APPID, 在评分的时候有需要这个ID)

在loader上传成功后, 过一会儿会在网页的创建版本处有个加号, 这时候就可以点击添加刚才上传的版本,

等这些都做好之后, 就可以点击提交审核了. 然后就是等待…

2018-12-23 18:09:29

记一次借贷类app的上传提审
创建应用
整个过程

  1. 首先需要在Appstore connect(之前叫iTunes Connect) 创建一个app, 这个可以一开始就创建, 获取app的id, 并且占一个坑
    1. 这时候需要填写一些信息: 名字 套装ID, SKU,
    2. 名字: 随便填, 最好就是app的名字
    3. 套装ID, 这个没有的话, 得先去账户中心创建一个, bundle ID 要和项目一样, 注意: wildcard那个不能使用推送还有一些服务, 注意看下自己需不需要, 一般的话我们创建specificId就可以了
    4. 回到connect页面, 等一会再刷新, 这时候套装ID就有选我们刚刚创建的那个了
    5. SKU单独标机这个app的标识符, 自己起一个就好了, 我是直接项目名加后缀
  2. 填好后就进入了这个应用的详情设置页面, 里面有更多的设置
  3. 根据信息填写好
  4. 最后的注意点是, 待提交的app版本需要和你开发的版本一致, 不然上传了也添加不进来的
  5. 其他的注意点随便找篇文章就可以

上传的图片, 需要是截图, 5.5寸的plus截图是必须要有的, 其他的类型, 默认是勾选使用5.5寸图片资源. 注意不要出现真实个人信息
这里我就直接用没有个人信息的截图了, 因为设计没有给, 之前的全是安卓的资源
在审核的下面, 可以上传一个附件, 录制你的测试视频, 提价上去, 据说让审核人员轻松点, 审核就块一点,
总之, 把它们当爸爸, 你就有可能过的越快

但是呢, 做到这样, 还是被拒绝了

为什么?

元数据拒绝
先不说应用本身怎么样吧. 因为上架的是借贷类型的产品, 苹果在拒绝的回复中提到要求提供信息的几个点. 并没有网上说的那些2.1拒绝的东西, 网上多是马甲包相关的, 因为这个是第一个上架的, 所以, 占时和那个无关.

这里给的回复, 几乎都是和你业务的资质, 以及业务开展, 产品利率以及期限等相关的东西.

呵呵…资质…这回也很难办了…

要回复的问题合补充的信息, 全是接待相关的问题. 等回复了, 再看看会有什么问题了
